第一部分: Data Modeler上机练习---小型数据库建模

本次上机将使用Data Modeler创建一个简单的图书馆数据库, 其中包括：
1. 图书实体(entities for books);
2. 图书馆主顾(patrons, people who have library cards);
3. 交易(transactions,借书checking a book out, 还书returning a book等).

  通过这里的练习，掌握基本利用Data Modeler创建Oracle数据库的基本用法

 

主要的练习内容:


1.  开发逻辑模型：Logical Model.
2.  开发关系模型：Relational Model.
3.  生成DDL.
4.   保存设计.

 

数据库创建上机过程:

1 开发逻辑模型：Logical Model

 
本次练习需要的逻辑模型有三个实体:
1. Books (描述图书馆里的每本书)；
2. Patrons (描述具有借书资格的每个人)；
3. Transactions (描述每次借还书过程:包括patron和book).

在创建实体前需要创建一些域，这些域将使实体entity的创建(包括后面的DDL生成)更有意义和特定性.

添加域：Adding Domains

 
创建数据库时, 可能要确定不同记录数据存放的类型及其范围, 就得对这些数据进行按域定义, 便于不同表间共享这些域所定义的数据类型, 比如这里的数据库需要定义下面的域:
1.  人名的: first name和last names,  可以多达25 characters.
2.  街道的:   Address lines， 可以多达40个characters.
3.  City名: 可以多达25个characters.
4. State代码: 可以用2-character的标准缩写.
5. Zip代码: 邮政编码, 可多达10个characters (nnnnn-nnnn).
6. Book identifiers: 可以用多达20个characters.
7. Numeric Id: 可以用7个数字(no decimal places).
8. Titles: 包括books, articles等, 可以多达50个characters.

你需要确定添加合适的域，这样你才可能在以后来用它们规定想要创建的实体属性的数据类型. (这些添加的域也将在退出Data Modeler以及重启后还可以再用.)

创建实体：Transactions

Attribute Name               Type                                       Other Information and Notes
1. transaction_id             Domain: Numeric Id              Primary UID (unique identifier). (Unique transaction ID number)
2. transaction_date         Logical type: Datetime          M (mandatory, that is, must not be null). Date and time of the transaction.
3. transaction_type          Domain: Numeric Id             M (mandatory, that is, must not be null). (Numeric code表示transaction类型, 如1表示借书.)

注: 不要显性定义patron_id 和book_id 属性, 因为这个在创建实体间联系后会自动加到Transactions entity (可以看睦Creating Relations Between Entities);

创建实体间关系

联系表示实体间的关系，分别有:
one-to-many:     1:N;
many-to-one:     N: 1;
many-to-many: M:N.

本例中存在以下的实体间的联系:
1. Books － Transactions间的联系: one-to-many. 每本书可以有多次交易记录.每本收有0次或1次借出交易; 每本借出的书不能再次被借出， 直到被还回来为止.

2. Patrons - Transactions间的联系: one-to-many. 每个借阅者可以有多个依次和同时的交易. 每个patron一次就可以借出一本或多本书, 并且可以有多个借书记录, 表示多次去图书馆;
每个借阅者可以随时返回所借书.

生成DDL

生成Data Definition Language (DDL)的语句，这样可以用来创建数据库对象，来反映前面设计的数据模型.

DDL语句将应用你规定的physical model (比如 数据库类型, 比如这里的Oracle Database 11g).
点菜单：File, Export, 选DDL File.

选数据库类型：　比如Oracle Database 11g，并点　生成.
生显示：DDL Generation Options对话框.

这里用所有的缺省值即可，点OK.

结果显示一个DDL编辑器：　其中有SQL语句，用于创建表，添加constraints.
(尽管可以编辑这些ＳＱＬ，但不要在这里编辑ＳＱＬ语句.)



点:  保存, 即可保存这些SQL语句到一个.sql脚本文件: 比如:  create_library_objects.sql.

随后可以SQL中运行这个脚本:
SQL>@create_library_objects.sql
结果就可以创建出想要的数据库了.

With the Relational_2 tab selected, click the Engineer to Logical Model icon.

You do not want the Address column added to the Sales subtype. Make sure the check box in front of Address is not checked.

 

Expand the EMPLOYEES node that corresponds with the Non-Sales entity and its columns in the Logical model. You do not want Address to be added to the Non-Sales subtype. Make sure the check box in front of Address is not checked.

 

Scroll up the list and expand the first EMPLOYEES node to see that the Address column will be added to the super type. Make sure the the check box in front of Address is checked. Then click Engineer.
